# Changelog

## v3.3
- Удалена возможность перепроходить игру заново (`restartGame`).
- Создано окошко меню после прохождения: теперь на финальном экране вместо кнопки перепрохождения доступна только кнопка просмотра таблицы лидеров.

## v2.8
- Исправлена логика регистрации: теперь одинаковые имена из одинаковых классов не могут регистрироваться (добавлена дополнительная проверка аккаунта).
- Уменьшена высота окна вопросов первого этапа (Викторины), чтобы игрокам не приходилось скроллить вниз для продолжения игры.

## v2.7
- Внедрена система "Умного сохранения"! Теперь прогресс игрока сохраняется в базу данных не только в самом конце игры, а **после прохождения каждого отдельного этапа**.
- Если игрок случайно закрыл вкладку или вышел из игры, при следующем входе система найдет его незавершенную сессию и восстановит все достижения.
- Общее время (`total_time`) теперь считается абсолютно честно (сумма времени каждого этапа), поэтому перерывы и выход из игры больше не будут портить статистику в таблице лидеров.

## v2.6
- В личном профиле игрока вместо процента потраченного времени (справа от прогресс-баров) теперь отображаются заработанные очки за каждый конкретный этап.

## v2.5
- Внедрены профили игроков! Теперь на любую строчку в таблице лидеров можно нажать, чтобы увидеть детальную статистику прохождения по этапам (с процентом потраченного времени и красивыми прогресс-барами).
- Добавлена новая плавающая кнопка «👤 Я» в верхнем левом углу для авторизованных пользователей. Она открывает ваш личный профиль профиль и статистику.
- В личный профиль добавлена кнопка «🚪 Выйти из аккаунта» с полным сбросом сессии (удалением из кеша) и возвратом на экран регистрации.

## v2.4
- Изменен алгоритм генерации почты: теперь русские символы транслитерируются (`petr_5a@ecoplayer.com`), что убирает "дичь со знаками процентов" и сохраняет уникальность аккаунта внутри класса.

## v2.3
- Кнопка «🏆 Лидеры» теперь доступна прямо на "Карте приключений" (рядом с кнопкой начала уровня), чтобы можно было смотреть рейтинг прямо во время игры.
- Кнопка "⬅️ Назад" в таблице лидеров теперь умная: она понимает, откуда вы её открыли, и возвращает вас именно туда (на экран входа или на карту).


## v2.2
- Таблица лидеров вынесена в отдельное красивое окно на весь экран (`screen-leaderboard`).
- Добавлена кнопка «🏆 Таблица лидеров» на экране регистрации, которая открывает этот рейтинг.
- Теперь данные лидеров загружаются не сразу при открытии сайта, а только при нажатии на кнопку, что экономит ресурсы интернета и выглядит более динамично.


## v2.1
- Добавлена живая "Таблица Лидеров" (Топ-10 игроков). Она отображается на стартовом экране под формой регистрации. Рейтинг автоматически сортируется сначала по наибольшему количеству очков, а затем по наименьшему затраченному времени.
- Добавлены стили для таблицы, включая иконки кубков и специальную подсветку топ-3 участников.


## v2.0 (Мажорное обновление)
- Полностью пересобрана система авторизации для обеспечения уникальности аккаунтов. Если игрок пытается зарегистрироваться с именем, которое уже занято в его классе, игра выдаст предупреждение и не пустит его (во избежание игры за чужой аккаунт).
- Теперь при проверке кеша `checkSession` обращается напрямую к серверу Supabase (`getUser()`), а не просто читает локальный токен. Это значит, что если аккаунт был удален вами вручную из базы, игра автоматически выбросит пользователя обратно на окно регистрации.


## v1.9
- Экран входа (`screen-auth`) теперь является единственным стартовым экраном. Если игрок уже есть в кеше — игра (карта) начинается моментально! Если нет — мы запрашиваем имя, и по клику на «🚀 Начать путешествие!» сразу стартует приключение.
- Улучшена адаптивность на телефонах (`max-width: 600px`): уменьшены огромные шрифты, отступы и карточки. Интерфейс теперь гораздо приятнее выглядит на маленьких дисплеях.
- Теперь аудио-подсказку от Эко можно **поставить на паузу**. Для этого достаточно нажать на самого Эко (персонаж справа) во время воспроизведения аудио.


## v1.8
- Исправлена ошибка инициализации `supabase` (`SyntaxError: Identifier 'supabase' has already been declared`), переменная переименована в `supabaseClient`.
- Полностью удалены скучные поля `email` и `password`.
- Реализован «бесшовный» вход: дети просто вводят «Имя», «Класс», «Букву», под капотом генерируется фейковая почта и универсальный пароль, что делает регистрацию и последующий вход молниеносными (нажатием одной кнопки "Начать игру").


## v1.7
- Добавлены поля «Отображаемое имя», «Класс» (1-11) и «Буква» (А-И) для формы регистрации. Выбор происходит через удобные `select`.
- Написана логика передачи и сохранения дополнительных метаданных пользователя (`display_name`, `grade`, `grade_letter`).
- Изменена логика сохранения прогресса: теперь имя и класс игрока тоже дублируются в таблицу `player_progress`, чтобы с результатами было удобно скомпилировать рейтинг.

## v1.6
- Создан новый обязательный стартовый экран `screen-auth` для авторизации и регистрации.
- Интегрирована база данных Supabase (Auth & Database) для агрегации аккаунтов.
- Добавлена логика автоматического входа через кеш с помощью `getSession()`.
- Реализовано отслеживание времени для каждого из 4-х этапов и общего времени игры.
- Добавлена автоматическая отправка результатов (очки и таймеры) в таблицу `player_progress`.

## v1.1
- Исправления багов и рефакторинг логики в `script.js`:
  - Исправлена озвучка подсказки: теперь `playStageHint()` правильно находит звук для финального экрана (`audio-stage5`).
  - Убраны «утечки» глобальных переменных в `window` (`_anagramCurrent`, `_anagramWord`); теперь используются строгие локальные переменные модуля.
  - Алгоритм сортировки в "Сопоставлении" (Matching) заменён на честный Fisher-Yates (`shuffleArray`) вместо старого ненадежного `.sort(() => Math.random() - 0.5)`.
  - Улучшен UX "Сопоставления": теперь выделение с термина/определения можно снять, если кликнуть по нему во второй раз.

## v1.0
- Глобальные изменения: полная перепись архитектуры.
- Разделение монолитного `index.html` на логические составляющие:
  - `style.css` для каскадных таблиц стилей.
  - `script.js` для логики приложения.
  - `index.html` очищен от встроенного кода, содержит только семантическую разметку и линки.
